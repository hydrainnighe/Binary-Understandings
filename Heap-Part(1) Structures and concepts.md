  Heap Structures and Concepts Part 1


stack‌ တွေက data တွေ ဆွဲသုံးရတာမြန်သလောက် Memory မှာ limit ထားတဲ့ size နဲ့ အသုံးပြုတာ ဖြစ်ပါတယ်။ 
ဒါပေမယ့် heap ကတော့ တခြားပုံစံတစ်မျိူးနဲ့လာပါတယ်။
heap တွေအကြောင်းကို သူတို့ ဘယ်လို ပုံစံတည်ဆောက် ထားသလဲဆိုတာ အရင်ကြည့်ရအောင်။

     +-----------------------+-----------------------+
     |  Prev Size (8 bytes)  |  Size + Flags (8b)    |  <-- Chunk Header (Metadata)
     +-----------------------+-----------------------+
     |                                               |
     |          User Data (Actual Content)           |  <-- malloc() returns this address
     |                (e.g., "ABCDEFG")             |
     |                                               |
     +-----------------------+-----------------------+
     |                Padding (Alignment)            |
     +-----------------------+-----------------------+

heap တွေမှာ data တွေကို chuck တွေအနေနဲ့ သုံးပါတယ်/ chuck တစ်တွဲချင်းစီမှာ Metadata နဲ့ Userdata ဆိုပြီးနှစ်ပိုင်းပါပါတယ်။ သူတို့နှစ်ခုကို ခြားပေးထားတဲ့ padding ဆိုတာလည်းရှိတာပေါ့။
metadata ထဲမှာ ဘာတွေပါလည်းဆိုရင် အဲ့ chuck ရဲ့ size ရယ် ၊ ရှေ့ chuck တွေရယ် ‌အခြေအနေ ( free ဖြစ်မဖြစ်) ကို ထည့်ထားပေးပါတယ်။

အကယ်၍ ကျနော်တို့က program က malloc သုံးပြီး data တစ်ချို့ ကို program မှာ အသုံးပြုဖို့အတွက် memory 10 bytes ခဏပေးပါဆိုပြီး kernal ကိုပြောလိုက်ပါတယ်။ 
တကယ်တော့ malloc က C မှာ memory management လုပ်တဲ့ Library Function ဖြစ်တဲ့အတွက် အဲ့ lib ကို အသုံးပြုပြီး kernal ကို လှမ်းတောင်းတဲ့ ‌သဘောပါ။

အဲ့ အခါ kernal က chuck တစ်ခု ထုတ်ပေးပါတယ်။ အဲ့ chuck က ကိုယ် တောင်းလိုက်တဲ့ bytes ထက်ပိုများတယ့် (64bit မှာဆို 16 နဲ့စားလို့ပြတ်တဲ့ total bytes , 32bit မှာဆို 8 နဲ့စားလို့ပြတ်တဲ့ )
total bytes တွေပါပါတယ်။(Memory Alignment လို့ခေါ်ပါတယ်)
	ဆိုတော့ ပြောရရင် chuck တစ်ခုမှာ 10 bytes တောင်းလည်း 24 bytes ဒါမှမဟုတ် 32 bytes ပြန်ထုတ်ပေးနိုင်ပါတယ်။
kernal က တောင်းထားတဲ့ bytes ထက်ပိုထုတ်ပေးထားပေးမယ့် ကျနော်တို့ တောင်းထားတဲ့ 10 bytes ကိုပဲ သုံးရမှာဖြစ်ပါတယ်။
(ဒီမှာ ထပ်ပြီး details ပြောရရင် kernal ကနေmemory ပလိန်းလိုက် အကြီးကြီးကို brk mmap သုံးပြီးတောင်းပါတယ်။ပြန် chuck တွေအဖြစ်ပိုင်းပေးရတာပါ)

ဘာလို့ ပိုထုတ်ပေးရလဲဆိုရင် metadata အတွက် byte အပိုထုတ်ပေးတဲ့သဘောပါ။
ပြီးတော့ metadata ရဲ့ အဆုံးကျမှ userdata တကယ့် သိမ်းတဲ့ data စလာပါတယ်။ 
(တစ်ချက် သိထားရမှာက heap က ‌chuck ထဲမှာ ဆိုရင်‌တော့ sequential အလိုက် သိမ်းတာပါ ဒါကြောင့် ရှေ့ဆုံး address သိရင်ကို data အတွဲလိုက်ကြီးကို ဆွဲထုတ် နိုင်ပါတယ်။)
	ပြီးတော့ အဲ့ userdata ရဲ့ ရှေ့ဆုံး address ကို malloc က pointer ထဲမှာ ထည့်ပြီးပါပြီ။ 

## How Heap Stores New Data

ကဲ structure တွေ သိပြီဆိုရင်တော့ သူတို့ ဘာတွေ ဘယ်လိုကြည့်ပြီးမှ data တွေစသိမ်းလည်းဆိုတာကို ပြောပါမယ်။

heap က data အသစ်တွေကို စသိမ်းတဲ့ အခါ option 3 ခုနဲ့လုပ်ပါတယ်။ တစ်ခု မရမှတစ်ခုပေါ့ ၊ 
နောက်ဆုံး သုံးခုလောက်နဲ့ မရဘူးဆိုရင်တော့ NULL အဖြစ်သတ်မှတ်လိုက်ပြီး သိမ်းမရတော့ပါဘူး ။

### Bin(Reuse)
	
ပထမဆုံး အဆင့်ကတော့ အရင် ရှေ့မှာ သုံးခဲ့တဲ့ chuck က ကိုယ် တောင်းတဲ့ size နဲ့ ကိုက်သလားကြည့်ပါတယ် ။ အဲ..ကိုက်ရင်တော့ အဲ့ chuck ကိုပဲသုံးပါတယ်။
	
### Top Chuck(Split)
	
အပေါ်က option ဖြစ်တဲ့ size နဲ့ ကိုက်မနေဘူးဆိုရင်တော့ အဲ့ chuck နဲ့ နောက်ဆုံး data ထည့်ထားတဲ့ (စပြီး လွတ်နေတဲ့ နေရာ) က နေ data ကို စထည့်ပါတယ်။
	
### Kernal(Extend)
	
အဲ အပေါ်က နှစ်ခုလုံးဖြစ်မနေဘူးဆိုရင်တော့ kernal က chuck အသစ်လုပ်ပေးဖို့ လှမ်းတောင်းပြီး အဲ့ chuck အသစ်မှာပဲ store လုပ်လိုက်ပါတယ်။

	ဒီသုံးဆင့်က အကြမ်းဖျင်းပြောတာပါ။ ထပ်ပြီး လေ့လာကြည့်ရအောင်။
		
## Bin (Reuse)
Details ကျကျဆိုရင် ပထမဆုံးအဆင့်မှာ ပြောတာကdata ထပ်ထည့်တော့မယ် ဆိုရင် သုံးခဲ့ပြီးသား size တူတဲ့ chuck ကိုရှာတယ်ပြောခဲ့ပါတယ်။
ဆိုတော့ သုံးခဲ့ပြီးသား chuck ကို ဘာကိစ္စ သုံးရတာလဲပေါ့။(အသစ်ထပ်ထုတ်စရာမလိုဘူးပေါ့ကွာ :) )(Reuse လုပ်တယ်ခေါ်ပါတယ်)
```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main() {
    char *ptr1 = (char *)malloc(10);
    if (ptr1 == NULL) return 1;

    strcpy(ptr1, "ABCDEFG");
    printf("Ptr1 Address: %p\n", ptr1);
    printf("Ptr1 Data: %s\n", ptr1);

    free(ptr1);
}
```
နားလည်အောင်ပြောရရင် program တစ်ခုက free(ptr) လုပ်လိုက်တဲ့အခါ အဲ့ chuck ထဲမှာ ရှိတဲ့ data က တကယ်ကြီး free သွားတာမဟုတ်ပါဘူး။
ဥပမာ ပြောရရင် program တစ်ခုက chuck ထဲမှာ ABCDEFG ထည့်ထား တယ် ပဲဆိုပါဆို့။
   program က free(ptr) ဆိုပြီးလုပ်တဲ့ အခါမှာ malloc က ဘာလုပ်လည်းဆိုရင် အဲ့ chuck ကို free ဖြစ်သွားပြီလို့ flag သတ်မှတ်ပေးလိုက်ပါတယ်။
(တကယ်တော့ အထဲမှာ data က ရှိတုံးပဲ)။ အဲ့တော့ နောက်တစ်ခါ program စလည်း data ကရှိတုံးပါပဲ။

	အဲ့မှာ နောက်တစ်ခုက bin ပါ ။
bin တွေအမျိုးမျိုးရှိပြီး အဓိက အလုပ်ကတော့ free(ptr) လုပ်ပြီးတဲ့ အခါ အဲ့ chuck ကို unallocated flag လုပ်ပစ်ပြီး bin ထဲမှာ သွားထည့်ပါတယ်။
program က လိုတယ်ဟေ့ဆိုပြီးပြောတဲ့ အခါ malloc က ဘယ်လောက်လိုတာ‌လည်းရှာရပါပြီ ။
အဲ့အခါ အသစ်တောင်းတဲ့ byte နဲ့ (greater than or equal >= ) ဖြစ်ရင် အဲ့ chuck ကို ပြန်သုံးရမှာပေါ့။ 
အဲ့အတွက် အရင် size ကို bin ကနေမှ ဆွဲထုတ် ရမှာပေါ့။ အိုခေ ကိုက်တယ်ဟေ့ဆိုရင် အဲ့ chuck ကို bin ကနေပြန်ထုတ်ပြီး allocated အဖြစ်ပြန်သတ်မှတ်လိုက်ပါတယ်။(ဒါပေမယ့် အရင်ဒေတာရှိနေသေးတုံးပဲ)။

အဲ့ကြောင့် new data ကို overwrite မလုပ်မချင်း အရင် ABCDEFG က chuck ထဲမှာရှိနေတုံးပါပဲ။
ဒီကိစ္စဘာလို့ပြောရလည်းဆိုရင် UAF(use after free) vuln က ဒီကိစ္စကြောင့်ဖြစ်တာဖြစ်လို့ပါ။
ဘာလို့ဆို program တွေမှာ အရင် data အဟောင်းတွေကို overwrite မလုပ်မချင်း ကြည့်နိုင်တာဆိုတော့ 🤧။

But bin လိုပြောတဲ့ အခါမှာ အမျိုးအစားအမျိုးမျိူးရှိပြီး chuck တွေ ကို free flag တွေ အဖြစ်သတ်မှတ်တဲ့ အခါ ဦးစားပေး ပုံချင်းကွာတာပါ။ 
ဥပမာ fast bin ဆိုရင်  bin တွေကို stack တွေပုံစံ ချိတ်ထားပြီး size မတူတာ ဘာညာဆိုရင် ကိုယ့် အရှေ့က bin တစ်ခုကို ပဲ ဆက်သွားကြည့်ရမှာပါ။
	ဘာ bin မှမရှိတော့ဘူးဆိုရင်တော့ NULL အဖြစ်သတ်မှတ်ပါတယ်။
(fast bin က အရေးသေးငယ်တဲ့ allocation တွေလုပ်တဲ့ အခါမှာသုံးတာပါ)

	တခြား bin တွေကို လည်း program execute လုပ်တဲ့ အခါမှာ သုံးပါတယ်။
small/large/unsorted စတဲ့ bin ကတော့ နှစ်ဖက်ကို သွားလို့ရတာဖြစ်တဲ့အတွက်ပိုပြီး memory space ပိုကျယ်ပါတယ်။ 
ဥပမာ program နောက်တစ်ခါ memory ထပ်တောင်းတဲ့ အခါ နောက်ဆုံး သုံးခဲ့ တဲ့ chuck မတူရင်တောင် ‌အဲ့ chuck  ရယ့် ရှေ့ရောနောက်က chuck တွေကို လှမ်းစစ်နိုင် သွားနိုင် သုံးနိုင်ပါတယ်။

  သိရမှာတစ်ခုက bin တွေဆိုတာလည်း chuck တွေကိုပြောတာပါပဲ ဒါပေမယ့် flag ကတော့ unallocated ဆိုပြီး သတ်မှတ်ထားတာပါ။ဒါကြောင့် memory ထဲမှာ 
ရှိနေတဲ့ သူတို့နေရာရော data တွေရောက ပြောင်းမသွားတာပါ။
	binတွေက တကယ်တော့ heap internal ကနေ chuck တွေရဲ့ metadata နဲ့ size တွေကို linked list ပုံစံ ချိတ်ပေးထားတာပဲဖြစ်ပါတယ်။
