# Heap Part - 2

    ဒုတိယအဆင့်ကို detail ကျကျ ပြန်‌ကြည့်ရအောင် ။
ရှေ့က bin ကိစ္စ‌တွေမှာ တောင်းတဲ့ size က bin တွေ ရဲ့ size နဲ့ enough ဖြစ်မနေတော့ရင် ဒုတိယ အဆင့် ကိုရောက်ပါပြီ။

ဒီအဆင့်မှာ brk နဲ့ mmap တွေရဲ့ အခန်းတွေပါလာပါတယ်။
ရှေ့ အဆင့်မှာ memory size မလောက်တာကြောင့် ဒီ အဆင့်မှာ malloc က  heap ရဲ့ နောက်ဆုံးက နေရာ အလွတ်ကြီးကို လှမ်းကြည့်ပါတယ်။
    program အများစုမှာ chuck တွေပြီးရင် heap ရဲ့ နောက်ဆုံးမှာ unallocated space ကြီးတစ်ခုရှိနေတတ်ပါတယ်။ 
Top chuck လို့လဲခေါ်သလို Reminder chuck လို့လည်းခေါ်ပါတယ်။

    (Wilderness) လို့လည်းခေါ်ပါတယ်။
အကယ်၍ ကျနော်တို့တောင်းတဲ့ size က အဲ့ space မှာလုံလောက်နေတယ်ဆို ရင် ပြဿနာမရှိပါဘူး။
malloc ကအဲ့ space ကနေ ခွဲထုတ်ပြီး ပေးလိုက်ရုံပါပဲ။
  
    (Wilderness မှာ 100KB ရှိတဲ့ဆိုပါဆို့ 10KB ပဲလိုရင် 10KB နဲ့ metadata နဲ့တခြား နည်းနည်းအတွက် ပဲဖြတ်ထုတ် ပေးပြီး 90KB ကို ဆက်သိမ်းပါတယ်။ပြောချင်တာက Wilderness က ကျုံ့ကျုံ့ သွားမှာပါ။)

 အဲ့ Top Chuck ကို new chuck တစ်ခုအဖြစ်ပြောင်းပြီး သုံးခွင့်ပေးလိုက်ပါတယ်။ 
  But အဲ့ free space ကြီးကရော request တဲ့ size အတွက် မလုံလောက်ဘူးဆိုရင် ဘာလုပ်ကျမလဲ။ 
mallocက kernal ကို sbrk သုံးပြီး  kernal ကို heap memory ပို ထပ်ထည့်ပေးဖို့တောင်းပါတယ်။
 အဲ့ဆို heap ကြီး ပို ကြီးသွားပြီး လုံလောက်သွားပြီပေါ့။

ဒါပေမယ့် heap ဆိုတဲ့ကောင်က နေချင်ရာမှာနေနေတဲ့ကောင် မဟုတ်ပါဘူး။ ဒီကောင်ရဲ့ အောက်မှာ libraries တွေ နဲ့ Stack ဆိုတဲ့ကောင်ကြီး အပြင် တခြား memory ထဲမှာရှိတဲ့ကောင် တွေနဲ့ တွဲပြီးနေရတာ ပါ။

ဒါကြောင့် sbrk က extend ခိုင်းလို့ heap ကြီးကြီးလာပေမယ့် သူ့ဘေးက memory အစိတ်အပိုင်းတွေနဲ့ collide ဖြစ်သွားရင် sbrk fail ပါတယ်။

    ဒါကို ဖြေရှင်းပေးမှာကတော့ mmap ပါ။
malloc က ဒီကောင်ကို သုံးပြီး  memory ထဲမှာ နောက်ထပ် sperated region တစ်ခု ထပ်ခွဲထုတ်ပြီးရင် အဲ့ထဲကို heap အသစ်တစ်ခု ထပ်သွားထား လိုက်ပါတော့တယ်။

ပြောရရင်
  ```text
|heap|libraries|stack|
```
ဆိုတဲ့ structure မှာ
```text
|heap|libraries|stack|new zone|
```
ဖြစ်သွားတာပါ။ 
ဒါ့ကြောင့် large space allocation တွေအတွက်ဆိုရင် heap ကရွေးချယ်စရာဖြစ်လာတာပါ။

mmap နဲ့ ခွဲထုတ်လိုက်တဲ့ New Region မှာ chuck တွေ အသစ် ဖန်တီးနိုင်တယ့် heap တစ်ခုပုံစံဖြစ်လာပါတယ်။

    sbrk mmap တွေကို system call တွေလို့ခေါ်ပါတယ်။

ဒါပေမယ့် အဲ့ mmap ကြီးကရော အသစ်ထပ် ခွဲထုတ်ဖို့ region မရှိတော့ဘူး OS memory full နေပြီဆိုရင် mmap လည်း fail ပါပြီး ၊ အဲ့အချိန်ရောက်ရင်တော့ NULL return ပြီး malloc လည်း ဘာမှမလုပ်နိုင်တော့ပါဘူး။
